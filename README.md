# 🏟️ Kick-off (UK)

A personal multi-sport fixtures board, in UK time. English fork of *Coup d'envoi*.
A GitHub Action collects fixtures, merges them into a single `matches.json`, and
the PWA reads that file locally (same origin → **no CORS, no API key in the browser**).

## Sources
- **Football (men)** — Premier League + World Cup 2026 + Euro 2028 via **openfootball** (public domain JSON, no key).
- **Women's football** — Women's Super League, UEFA Women's Champions League, FIFA Women's World Cup 2027 via **TheSportsDB** (free key, called server-side). Best-effort coverage.
- **Rugby** — Six Nations via TheSportsDB.

## Repo layout
```
.
├── index.html                       ← the PWA
├── build_data.py                    ← the collector
├── matches.json                     ← generated by the Action
├── icon.png                         ← 180×180 home-screen icon
└── .github/workflows/update-fixtures.yml
```

## Setup (5 min)
1. Create a GitHub repo, push `index.html`, `build_data.py`, `matches.json`, `icon.png` and the `.github/workflows/` folder.
2. **Settings → Pages**: source = branch `main`, folder `/ (root)`.
3. **Settings → Actions → General → Workflow permissions**: enable **Read and write permissions**.
4. **Actions tab → Update fixtures → Run workflow** for a first fill. Then it runs twice a day.
5. Open the Pages URL → on iPhone, “Add to Home Screen”.

> Note: Premier League **2026-27** isn't published by openfootball yet (released mid-June), so upcoming PL fixtures appear automatically once it's added — nothing to change. Women's competitions depend on TheSportsDB coverage.

## Adding a competition later
- **openfootball league**: add `(name, code, timezone)` to `OPENFOOTBALL_LEAGUES` (e.g. `("Championship", "en.2", "Europe/London")`).
- **TheSportsDB**: add `(name, [name substrings], sport, label)` to `TSDB_SOURCES`. League id is resolved automatically by name.
